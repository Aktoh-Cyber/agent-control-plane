## Actual ESLint Issues in Codebase

While there are **zero suppressions** in source code, running ESLint reveals the following **warnings** (not errors) that should be addressed:

### Issue Summary by Category

| Category                      | Count | Severity | Action Needed                   |
| ----------------------------- | ----- | -------- | ------------------------------- |
| Unsafe type operations        | ~70   | Warning  | Improve type safety             |
| Unexpected console statements | ~15   | Warning  | Replace with structured logging |
| Unused variables              | ~5    | Warning  | Remove or prefix with \_        |
| Explicit any types            | ~5    | Warning  | Add specific types              |

### Priority Recommendations

#### 1. Replace console statements with structured logging (P1)

**Issue:** 15+ console.log/warn/error statements in source files
**Solution:** Use the Winston logger infrastructure (already implemented in P1)

```typescript
// ❌ Current
console.log('Starting server...');

// ✅ Recommended
import { logger } from '../utils/logger';
logger.info('Starting server...');
```

#### 2. Improve type safety for error handling (P2)

**Issue:** ~70 unsafe type operations, mostly in error handling
**Solution:** Use typed error classes from the error hierarchy (already implemented in P2)

```typescript
// ❌ Current
catch (error) {
  console.log(error.message); // Unsafe
}

// ✅ Recommended
import { ApplicationError } from '../errors';
catch (error) {
  if (error instanceof ApplicationError) {
    logger.error(error.message);
  }
}
```

#### 3. Remove unused variables or mark as intentional (P3)

**Issue:** 5 unused variables triggering warnings
**Solution:** Prefix with underscore or remove

```typescript
// ❌ Current
function handler(req, res, next) { // 'next' unused

// ✅ Recommended
function handler(req, res, _next) { // Marked as intentionally unused
```

#### 4. Replace explicit 'any' types (P3)

**Issue:** 5 explicit any type declarations
**Solution:** Use specific types or generics

```typescript
// ❌ Current
function process(data: any) {

// ✅ Recommended
function process<T>(data: T) {
// OR
function process(data: unknown) {
```
